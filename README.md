## Лабораторная работа №1
_Разработка графического пользовательского интерфейса (GUI) для языкового процессора_

### Цель работы
Разработать оконное приложение - текстовый редактор с графическим интерфейсом пользователя, который в дальнейшем может быть расширен до полноценного языкового процессора для анализа исходного кода.

Год разработки: 2026_

### Описание проекта
В рамках лабораторной работы реализовано настольное приложение «Text editor» - специализированный текстовый редактор с графическим интерфейсом пользователя. Программа предназначена для создания, открытия, редактирования и сохранения текстовых файлов, а также содержит отдельную область для вывода результатов работы языкового процессора.

Интерфейс приложения состоит из основного меню, панели инструментов, области редактирования текста и области отображения результатов. Пользователь может изменять размеры окна приложения и соотношение размеров рабочих областей.

На текущем этапе реализован базовый функционал текстового редактора. Пункты меню «Текст» и команда «Пуск» предусмотрены архитектурой программы и будут реализованы на следующих этапах разработки.

### Используемые технологии
Язык программирования: C#\
Фреймворк для GUI: Windows Forms\
Платформа: Microsoft Windows\
Среда разработки: Microsoft Visual Studio 2022\
Формат сохраняемых файлов: .txt

### Инструкция по сборке и запуску

#### Системные требования
Операционная система: Windows 10/11 (x64)\
Для запуска готового приложения: дополнительное ПО не требуется\
Для сборки из исходного кода: .NET 7.0 SDK

#### Запуск готового приложения
Загрузить архив с последней версией из раздела Releases репозитория.\
Распаковать архив в любой каталог.\
Запустить исполняемый файл TextEditor.exe.

#### Сборка из исходного кода
Установить .NET 7.0 SDK.\
Клонировать репозиторий:

> git clone <URL_репозитория>

Выполнить сборку:

> dotnet build -c Release

Перейти в каталог с собранным исполняемым файлом:

> cd bin\Release\net7.0\

Запустить приложение командой:

> TextEditor.exe

#### Область редактирования
![Область редактирования](screen/text.png)

Верхняя часть окна представляет собой текстовый редактор на основе RichTextBox. В этой области пользователь вводит и редактирует текст. Все команды меню «Файл» и «Правка» применяются к содержимому данной области.

![Область отображения результатов](screen/grid.png)

Нижняя часть окна предназначена для отображения результатов работы языкового процессора. На текущем этапе область реализована в виде таблицы DataGridView и доступна только для чтения.

### Ограничения и особенности реализации
- Подсветка синтаксиса и анализ текста не реализованы на текущем этапе.
- Пункты меню «Текст» и команда «Пуск» являются заделом для последующих лабораторных и курсовой работы.
- Поддерживается работа с одним файлом одновременно.

### Постановка задачи
Разработать лексический анализатор (сканер) в соответствии с индивидуальным вариантом задания, интегрировать его в приложение из лабораторной работы №1 и обеспечить наглядный вывод результатов.

#### Требования к реализации сканера:

Спроектировать диаграмму состояний конечного автомата, реализующего сканер, согласно варианту задания.\
Разработать программный модуль лексического анализа, который:\
принимает на вход строку (исходный текст программы);\
выделяет все допустимые лексемы согласно варианту;\
классифицирует лексемы по типам (ключевое слово, идентификатор, число, оператор, разделитель и т.д.);\
любые символы, не соответствующие ни одному из допустимых типов лексем, считать недопустимыми и выводить сообщение об ошибке с указанием позиции;\
учитывает многострочность входного текста.\

#### Требования к интеграции и интерфейсу:

Встроить сканер в ранее разработанный интерфейс и связать его с кнопкой «Пуск» и соответствующим пунктом меню.\
Область вывода результатов (элемент DataGridView) должна содержать столбцы:\
Условный код (числовой идентификатор типа лексемы);\
Тип лексемы (текстовое описание);\
Лексема (выделенная подстрока);\
Местоположение (номер строки, начальная и конечная позиция символов).\
Реализовать навигацию по ошибкам: при щелчке на сообщении об ошибке в таблице результатов курсор в области редактирования должен устанавливаться на позицию недопустимого символа.\
